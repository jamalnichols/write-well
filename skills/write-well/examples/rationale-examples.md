# Rationale Examples

## Good rationale

### Revision returned

We cut the launch memo to one clear claim and moved the real news to the end of the first sentence.

### Why I rewrote it this way

- Cut the throat-clearing so the memo gets to the point faster.
- Replaced abstract nouns with direct actions, which made the prose sound less managed.
- Reordered the first two sentences so familiar context comes first and the actual news lands with more force.
- Preserved the original skepticism instead of neutralizing it into bland professionalism.

## Another good rationale

- Trimmed filler phrases that made the paragraph sound AI-written.
- Turned buried actions into verbs so the claims felt more direct.
- Strengthened sentence endings so the important ideas land harder.
- Left the sharper phrase alone because it carried the writer's voice.

## Bad rationale

- Improved readability.
- Enhanced clarity.
- Optimized the flow.
- Made it more engaging.

Why this is bad:

- too generic
- could describe any edit
- does not name real changes
- sounds like the prose the skill is supposed to remove

## Rule for rationale quality

A good rationale should point to a real editorial move, not a vague benefit claim.
